# AI 视频人物动作提示词（图生视频通用）

> **性质**：社区/从业者总结，**非**即梦/Seedance 官方文档。  
> **画布字段**：`storyboardShots[].videoMotionPrompt` 或 `videoNode.data.video.draft.prompt`（动态层，在 `visualPrompt` 静态画面之上）。  
> **Hermes 工具**：`film.shot_to_video_prompt`（由 visual 生成初稿后，可按本模板润色）；批量出视频前务必有明确动作描述。  
> **参数真源**：[`models/seedance-params.md`](../models/seedance-params.md)

## 核心原则

AI 视频提示词**不是越长越好**，也**不是动作越多越好**。

| 写法 | 结果 |
|------|------|
| 只写小动作（「抬眼」「手指收紧」） | 人物动得太少 |
| 动作堆叠（走+转+抬手+低头+回望+后退…） | 画面乱、抓不住重点 |

**稳定结构（图生视频推荐）**：

**1 个主动作 + 1 个辅助动作 + 1 个情绪状态 + 1 个结束画面 + 1 句镜头描述**

## 万能基础模板

```
【人物身份】位于【场景环境】中，整体氛围【场景气质】。人物当前状态是【情绪状态】。

画面开始时，人物先【起始状态】，随后完成【主动作】。在动作过程中，加入【辅助动态】，让【身体部位】随着动作产生自然联动。

最后人物停在【结束状态】中，情绪保持【最终情绪】。镜头【简单镜头描述】，画面保持【整体风格】。
```

**示例（古风行走）**

黑衣古风女子站在红色柱子旁，暖色灯笼在背景中柔和发光，整体氛围安静克制。人物当前状态是冷淡、疏离。

画面开始时，她先短暂停顿，随后缓慢向前走去。经过柱子时，身体轻轻侧过，衣袖和发丝随着动作自然摆动。

最后她慢慢停下，抬眼看向侧前方，神情依旧安静冷淡。镜头从侧面缓慢跟随，保持中近景，画面保持古风电影感。

## 步态模板（主动作常为「走」）

步态决定身份气质；同样是走路，古风女子、侠客、宫廷、受伤、紧张角色写法应不同。

**通用**

```
【人物身份】从【起始位置】开始向【方向】走去。她/他的步伐【步态特点】，行走速度【快慢程度】，身体状态【平稳/轻盈/沉重/紧张/克制】。整体动作符合【人物身份或场景气质】。
```

| 类型 | 适用 | 要点 |
|------|------|------|
| 端庄 | 宫廷、贵族 | 步伐沉稳、速度慢、含蓄克制 |
| 谨慎 | 夜街、危险 | 放轻、慢、观察环境、警觉 |
| 压迫 | 反派、高手逼近 | 每步很稳、肩略下压、靠近目标 |
| 疲惫 | 受伤、逃亡后 | 沉重、肩垂、动作更慢 |
| 轻快 | 少女、明亮场景 | 有节奏、衣发自然摆动 |

## 辅助动态（只选一个）

在主动作后补**一条**辅助动态，避免堆叠。

常用短句：衣袖随动作轻摆；发丝轻微晃动；耳坠发饰细晃；手指收紧又放松；眼神短暂扫向一侧；经过柱子时身体轻侧；抬手整理袖口；停步时衣摆延迟轻晃。

```
在【主动作】过程中，人物加入【辅助动作】，同时【衣袖/发丝/饰品/手指/眼神】产生自然变化，让动作更完整，但整体不要夸张。
```

## 身体联动

避免只有单个部位在动。

```
人物完成【主动作】时，【头部/肩膀/上身/手臂/手指】随动作产生自然联动。动作幅度保持【克制/明显/自然】。
```

| 联动 | 写法要点 |
|------|----------|
| 抬头 | 视线上移，肩微变，上身稳定带动 |
| 转头 | 眼神先偏，头再转，肩上身小幅度跟随 |
| 抬手 | 腕先动、指展开、肩带手臂、衣袖下垂摆动 |
| 抱物 | 身体侧转时手臂调位、指收紧、袖褶变化 |

## 情绪状态

动作是「做什么」，情绪是「为什么」。

```
人物当前情绪是【情绪状态】。这个情绪通过【眼神/表情/呼吸/手部/姿态】表现出来。动作不需要夸张，但要让人感受到内心状态。
```

| 状态 | 关键词 |
|------|--------|
| 压抑隐忍 | 指收紧、下颌绷紧、呼吸沉、眼神沉淀 |
| 警觉 | 眼神快扫、肩收紧、转向声源 |
| 冷静疏离 | 动作慢、无表情剧变、目光远 |
| 决绝 | 眼神坚定、肩下沉、已做决定 |
| 紧张 | 目光不安、肩收紧、移动更谨慎 |

## 动作链（约三步）

**起始状态 → 主动作 → 辅助动作 → 结束状态**（最多再拆 1～2 步，勿超过三步主链）

| 链类型 | 结构 |
|--------|------|
| 行走 | 停顿 → 向前走 → 经过物体侧身 → 停步抬眼 |
| 回头 | 停顿 → 头转向后 → 肩上身跟随 → 视线落点 |
| 转身 | 停顿 → 偏头 → 肩转 → 完成转身定神 |
| 起身 | 抬眼 → 手按桌沿前倾 → 缓缓站起 |
| 靠近 | 目光锁定 → 稳定靠近 → 停在近处 |
| 后退 | 眼神变化 → 重心后移半步 → 重新站稳 |

## 多人画面（分角色写）

**不要**混写：「一男一女在房间里对视」。

**要**写清：谁先动、谁回应、互动关系、结束关系。

```
【场景】中，【角色A】位于【位置】，【角色B】位于【位置】，关系【关系状态】。

角色A先【主动作】，情绪【A情绪】。
角色B随后【回应动作】，情绪【B情绪】。
两人【互动关系】，最后停在【结束画面】。镜头【镜头描述】，重点【关系重点】。
```

对峙 / 告别 / 保护 / 争执：分别写 A 主动、B 回应、手部与距离、结束对峙或挽留状态。

## 镜头短句（结尾一句即可）

| 场景 | 镜头 |
|------|------|
| 向前走 | 侧面跟随 / 正面缓慢后退 |
| 情绪变化 | 缓慢推进面部 |
| 离开 | 背后跟拍 |
| 对峙 | 中近景轻微推进 |
| 独处 | 固定机位 |
| 气场强 | 轻微环绕 |

示例：镜头保持中近景轻微推进，重点捕捉眼神变化。

## 防「不动」与防「乱」（可贴在文末）

**动得太少时补一句：**

- 人物需要完成明确的连续动作，不要只站立不动，不要只做轻微表情变化。
- 动作需包含明显身体移动和结束姿势，不要停在准备阶段。

**动作太乱时补一句：**

- 动作保持简洁清晰，不要加入多余动作，不要频繁切换姿势。
- 整体控制在一个主动作和一个辅助动作内，保持画面稳定。

## 最终可复制总模板

```
【人物身份】位于【场景环境】中，整体氛围【场景气质】，人物当前状态是【情绪状态】。

画面开始时，人物先【起始状态】，随后完成【主动作】。在动作过程中，加入一个【辅助动态】，例如【衣袖摆动/手指收紧/轻轻侧身/整理袖口/眼神短暂移动】。

随着动作发生，人物的【头部/肩膀/上身/手臂/手指】产生自然联动，不要只有单个部位在动。

最后人物停在【结束状态】中，情绪保持【最终情绪】。镜头【简单镜头描述】，重点突出【画面重点】。

人物需要完成明确动作，不要只站立不动，不要只做轻微表情变化；动作保持简洁清晰，不要加入过多无关动作。
```

## 单人完整模板

```
【人物身份】站在【场景】中，周围是【环境元素】，整体氛围【安静/紧张/压抑/华丽/神秘】。

她/他当前状态是【情绪状态】，神情【具体神态】，目光【视线方向】。

画面开始时，人物先【短暂停顿/低头/站定/看向前方】，随后【主动作：向前走/转身/抬头/靠近/后退/起身】。

动作过程中，加入一个辅助动态：【轻轻侧身/整理衣袖/手指收紧/抬手触碰物品/眼神扫向一侧】。

身体的【肩膀/上身/头部/手臂】随着动作自然联动，【衣袖/发丝/饰品】产生轻微晃动。

最后人物【停步/抬眼/转头/站定/看向目标】，情绪停在【克制/警觉/决绝/疏离/压抑】状态中。

镜头【缓慢推进/侧面跟随/保持中近景/轻微环绕】，重点突出【人物表情/动作过程/眼神变化/人物气质】。

人物需要完成明确的动作过程，不要只做轻微表情变化；动作保持简洁，不要加入过多无关动作。
```

## 多人完整模板

```
【场景】中，【角色A】位于【左侧/前景/中心】，【角色B】位于【右侧/后方/近处】，两人距离【很近/一步/面对面】，关系【对峙/争执/保护/告别/试探】。

角色A先【主动动作】，情绪【压迫/冷静/克制/愤怒/犹豫】。
角色B随后【回应动作】，情绪【紧张/倔强/不安/克制/震惊】。
两人【互动关系】，最后停在【结束关系】。

镜头保持【中近景/轻微推进/固定】，重点突出【两人距离/眼神/手部/关系】。

多人动作要分清谁先动、谁回应；不要只站着对视。
```

## 与 CanvasFlow 字段

| 字段 | 用途 |
|------|------|
| `visualPrompt` | 静态画面：景别、环境、光影、服饰 |
| `videoMotionPrompt` / `draft.prompt` | **动态层**：主动作、辅助动态、联动、情绪、镜头 |
| 项目圣经 `visualStyle` | 风格锚词与 `visualPrompt` 一致 |

## Hermes 如何触发（对话驱动，无需点技能）

用户对灵体说即可，例如：「帮各镜写人物动作视频提示词」「第 2 镜改成缓慢向前走再出视频」「批量出视频」。

导演自动行为：

1. 识别话术 → 计划步骤 `film.shot_to_video_prompt`，`args.useMotionTemplate=true`
2. 检索本文档（RAG）+ 调用对话模型，按模板为每镜写入 `videoMotionPrompt` 与 `video.draft.prompt`
3. 若用户接着说「出视频」，会在 `video.generate_for_beats` **之前**自动先补全动作提示词

实现：`filmCharacterMotionPrompt.ts` + `hermesPlanFromIntent` 规则；侧栏不展示「人物动作」技能芯片。

## 资料总结（一句话）

先定身份与场景气质，再安排**一个主动作、一个辅助动态、一个情绪、一个结束画面**；单人用动作链，多人分角色写谁先动谁回应；结尾加一句镜头；必要时加防不动/防乱约束句。
