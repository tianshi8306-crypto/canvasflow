# LibTV 操作指南与 CanvasFlow（vibevideo）方案对齐

> **目的**：把 LibTV 官方使用指南（你提供的 `D:\libtv\LibTV使用指南_*.png` 共 57 页）里的**能力树**固化为本仓库的**产品北极星**，与现有 `ROADMAP_V2.md` 的 R1–R9 对照，避免后续开发在「无限画布 + 节点工作流 + AI 生产链路」上越偏越远。  
> **定位说明**：本仓库目标是 **本地单机、工程与数据以本机为主**；LibTV 为 **云端账号 + 会员/积分 + 多模型托管**。对齐的是 **交互范式与创作链路**，不是逐 API 复刻商业后台。

---

## 1. 产品北极星（一句话）

**LibTV 式体验**：无限画布上，用「文本 / 图片 / 视频 / 音频 / 脚本」五类节点搭工作流，配合底部或侧栏的「生成器」，完成从剧本/脚本 → 分镜图 → 分镜视频 → 剪辑合成的连续创作；并具备项目菜单、左侧工具栏、小地图、节点右键与批量能力等画布基础设施。

**CanvasFlow 差异约束（必须写进评审）**：

- **本地真源**：`canvasflow.json` + `assets/` + `.canvasflow/runs.db`；API Key 不进工程文件。
- **壳技术栈**：Tauri + WebView + React + XYFlow（不是 Lib 的网页壳本身）。
- **能力交付**：以 R 轮小步为准；本表中的「Lib 已有」在本地可以是 **长期目标态**，须在迭代单写清「本轮不做」。
- **生成器形态（2026-05-18）**：**不采用** LibTV 式底部 Dock；图片 / 视频 / 音频生成参数在 **选中节点时的节点外浮层**（如 `ImageGenerationPanel` Portal）+ **Inspector** 绑定元数据。详见 [`图片节点设计.md`](../图片节点设计.md)、[`SHORTCUTS.md`](./SHORTCUTS.md)。

## 2. 指南目录结构（据指南图 01–03 归纳）

下列编号与 Lib 指南章节对应，便于你我对照原图页码。

### 2.1 无限画布

| 小节 | LibTV 能力摘要 |
|------|----------------|
| 1.1 新建画布 | 首页「开始创作」、项目列表与「全部项目」入口；进入画布 |
| 1.2 五大基础节点 | 文本 / 图片 / 视频 / 音频 / 脚本；新建方式含双击空白、拖入媒体、右键菜单 |
| 1.2.1 新建节点 | 双击画布、拖入文件、添加资源（上传 / 图库） |
| 1.2.2 文本节点 | 手写与「文本生成器」大模型生成；节点内菜单入口 |
| 1.2.3 图片节点 | 上传与图模生成；选中节点上浮层工具栏（高清、扩图、多角度、打光、重绘等） |
| 1.2.4 视频节点 | 上传与视频模型；底部「视频生成器」多 Tab（文生视频、图生视频、参考等）；`@` 引用素材 |
| 1.2.5 音频节点 | 上传波形预览；底部「音频生成器」（TTS / 音乐 / 音效等） |
| 1.2.6 脚本节点 | 上传剧本 / 角色图+剧情 / 参考视频生成分镜脚本；卡片视图与**全屏表格**编辑；接分镜图 |
| 1.2.7 节点操作 | 右键：存素材、创建主体、复制粘贴删除等；悬停工具条；**副本**保留连线 vs 复制不保留连线 |
| 1.3 工作流搭建 | 多节点连线、一键执行类能力；Showcase 示例 |
| 1.4 四大画布功能 | 项目菜单栏、左侧栏、个人中心、**小地图** |

### 2.2 画布实用工具（Slash 等）

| 小节 | LibTV 能力摘要 |
|------|----------------|
| 2.1 Slash 快捷 | 多机位九宫格、剧情推演四宫格、25 宫格连贯分镜、电影级光影矫正、角色三视图、画面推演（前/后几秒）等 |
| 2.2 图像工具 | 720° 全景、多角度、打光、九宫格、基础编辑、宫格切分、标注、旋转镜像 |
| 2.3 视频工具 | 视频高清、解析、剪辑、合成 |

### 2.3 生成器与模型

| 小节 | LibTV 能力摘要 |
|------|----------------|
| 3.x 图像生成器 | 风格、焦点编辑、镜头聚焦、摄像机控制等 |
| 4.x 视频生成器 | 视频主体库等 |
| 5.x 模型清单 | 图像 / 视频 / 语言 / 音频模型及说明（含各厂商能力表、合规素材库等） |

### 2.4 其它

- **快捷键**：指南后部章节（原图约 50 页以后）— 实现时须另建 `docs/product/SHORTCUTS.md` 与 App 内一致。
- **真人/合规素材库**（如 Seedance 2.0 说明）：强依赖云端审核与法务；**本地版默认列为「可选远期 / 或仅 UI 占位」**，除非单独立项。

## 3. 能力对照表（LibTV ↔ 当前仓库 ↔ 建议轮次）

**图例**：`已有` 可用主路径 / `部分` 有雏形或仅一种路径 / `缺口` 未做或明显弱于 Lib / `目标` 明确在路线图中但未到交付

| Lib 能力域 | 当前 CanvasFlow（截至文档编写时） | 建议归属（四层 / R 轮） | 备注 |
|------------|-------------------------------------|-------------------------|------|
| 无限画布 + 缩放平移 | `已有`（XYFlow + 小地图 + F/Z 聚焦） | 创作体验 R2 | 双击新建、框选、整理画布见 `SHORTCUTS.md` |
| 五类基础节点 | `部分`（含脚本、媒体路径、LLM） | 创作体验 R2 | 与 Lib 的「节点内子菜单 / 底部生成器」布局不同，属**形态差异**，功能可对齐 |
| 拖入媒体建节点 | `已有` | 创作体验 R2 | |
| 脚本工作台（表/卡、勾选） | `部分` | 生产链路 R3 | 已有表/卡与勾选；**全屏表格**已对齐 Lib 子集（`iteration-06`）；角色图/视频入格、字段显隐与键盘流已补齐，字段显隐方案尚未支持模板级共享 |
| 分镜文案（LLM） | `已有` | 生产链路 R4 | Lib 下一步为**分镜图网格 + 批量出图**：缺口，接 R4/R5 |
| 分镜图生成 / 批量 | `部分` | R4–R5 | 已支持**本机选图**导入 `assets/` 并关联预览（`iteration-05`）；云端文生图 / 批量出图仍为缺口 |
| 分镜图 → 视频 / 批量视频 | `缺口` | R5–R6 | 对应 Lib「批量生成视频」、时间线 |
| 底部图像/视频/音频生成器 | **不对齐**（节点外浮层 + Inspector） | 创作体验 R2/R5 | 已决策不引入底部 Dock；能力对齐 Lib 生成器，形态为浮层 |
| 节点上浮层（高清、扩图、解析…） | `部分`（图片已交付；视频顶栏 UI+stub+下载，见 `video-node-chrome-phase2.md`） | R5 / R8 | 与媒体特性、资产库联动 |
| Slash 快捷工具全家桶 | `缺口` | R8 或独立 Epic | 易膨胀；建议按「最高频 3–5 条」切片 |
| 工作流 DAG 执行 | `部分`（拓扑执行 LLM/FFmpeg） | 能力编排 R7 | scriptNode 在图中执行逻辑与 Lib 不同，可演进 |
| 我的资产 / 主体库 | `部分`（AssetPanel、DB 索引） | 资产与质量 R8 | Lib 的「创建主体、合规库」更深 |
| 项目菜单 / 全部工程 | `部分` | R1 / R9 | 本地文件夹即工程 |
| 小地图 | `已有` | R2 | `FlowCanvas` MiniMap + `Alt+Shift+M` |
| `@` 引用节点进提示词 | `部分` | R5–R7 | `MentionInput` 已用于 LLM/图片浮层；参考图仍以连线为准 |
| 会员 / 积分 / 云端-only 模型 | **不对齐** | — | 本地改为「自备 Key / 本地模型 optional」 |

## 4. 防偏离的落地流程（团队约定）

1. **新开迭代前**：执行单里写「本轮对齐 Lib 指南章节 x.x（若完全无关则写：本轮不涉及 Lib 对齐）」。
2. **设计争议**：先查本表与 `docs/iterations/ROADMAP_V2.md` 边界；仍争议则更新本文件「决策记录」一小节（日期 + 结论）。
3. **大功能（底部生成器、Slash、主体库）**：禁止静默混入小迭代；须独立 R 或 Epic，并更新对照表状态列。
4. **指南更新**：若你替换 `D:\libtv` 图片或增页，在本文顶部「修订记录」记一行日期与变更摘要。
