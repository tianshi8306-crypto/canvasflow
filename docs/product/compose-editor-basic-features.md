# 剪辑工作台 — 基础视频编辑功能清单

> 对照：图一 = 当前实现；图二 = 目标体验（CapCut / 即梦类单轨时间线 + 片段右键菜单）。  
> 本文只整理**基础剪辑**应具备的能力与现状，不展开多轨特效、AI 人声分离等进阶项。

---

## 一、产品定位（一句话）

**画布节点「剪辑」= 多段成片的时间线编排器**：从上游视频节点收集素材 → 在时间线上排序、裁切、预览 → FFmpeg 导出一条成片。  
不是完整 NLE（Premiere），但交互应对齐「上预览 + 下时间线 + 片段级操作」的最低专业线。

---

## 二、图一问题归纳（为何显得不专业）

| 现象 | 根因 |
|------|------|
| 标题仍显示「视频合成」 | 工程里节点 `data.label` 已持久化旧名；默认值改了也不会自动改旧工程 |
| 时间线大片空白、单轨很薄 | 底部区域 `max-height: 42vh` 固定，轨道高度未按「工具栏 + 刻度 + 轨」比例分配 |
| 片段像深色竖条、看不清画面 | 胶片条用同一段 `<video>` 重复铺满，未做多帧缩略图；加载失败时只剩占位色 |
| 工具栏有图标但能力不对齐 | 部分按钮（撤销/分割等）**未接逻辑**或藏在「更多」里，与图二主工具栏不一致 |
| 功能「像播放器不像剪辑器」 | 数据模型只有 `inputs: string[]` 路径列表，**无入点/出点/分割结果** |

## 三、基础功能分层（建议实现顺序）

### P0 — 没有就不算「能剪辑」

必须具备 **数据 + 后端 + UI** 闭环。

| # | 功能 | 用户动作 | 数据/后端要求 | 现状 |
|---|------|----------|---------------|------|
| 1 | **时间线浏览** | 横向滚动、缩放、刻度尺 | `duration`  per clip；`pxPerSec` | ⚠️ 有刻度/缩放；缩放与滚动体验待抛光 |
| 2 | **播放头** | 拖动/点击定位；预览同步 | 全局 `playheadSec` ↔ 片段内 offset | ✅ 含磁吸到片段边界 |
| 3 | **预览播放** | 播放/暂停、空格键 | 单 `<video>` 预览当前片段或成片 | ✅ 有；缺全屏 |
| 4 | **片段列表** | 从连线收集、显示顺序 | `inputs[]` + `collectClipsFromEdges` | ✅ 有 |
| 5 | **片段选择** | 点击片段高亮 | `selectedIndex` | ✅ 有；选中样式需对齐图二（白框 + 胶片） |
| 6 | **片段排序** | 拖拽调整顺序 | 重排 `inputs` | ✅ 拖拽有 |
| 7 | **片段删除** | 删除选中片段 | splice `inputs` | ✅ 工具栏有 |
| 8 | **片段裁切（入/出点）** | 修剪入点/出点工具 | `timelineClips` + FFmpeg trim | ✅ 迭代 20 |
| 9 | **在播放头分割** | 剪刀工具：一片变两片 | 分割后更新 `timelineClips` | ✅ 迭代 20 |
| 10 | **顺序预览（连播）** | 从当前片段播到结尾 | 段末 `ended` → 跳下一段 | ✅ 工具栏主区 |
| 11 | **导出成片** | 顶栏导出 | `render_timeline` → trim + concat | ✅ 迭代 19 |
| 12 | **撤销 / 重做** | Ctrl+Z / 工具栏 | 时间线操作栈（独立于画布 undo） | ✅ 迭代 20 |
| 13 | **片段缩略图（胶片）** | 轨道上可见多帧画面 | 每格按 in/out seek 不同帧 | ✅ 迭代 21 |
| 14 | **片段右键菜单** | 删除、分割、打开源节点 | 与图二一致的最小集 | ✅ 迭代 21 |

### P1 — 基础体验完整（对齐图二主工具栏）

| # | 功能 | 说明 | 现状 |
|---|------|------|------|

| 16 | **变速** | 片段 0.5x–2x（导出重编码） | ❌ |

### P2 — 明确不做进「基础剪辑」（单独迭代）

| 功能 | 说明 |
|------|------|
| 多轨视频/音频 | 需时间线引擎 + 混音 |
| 转场 / 滤镜 / 字幕轨 | NLE 级 |
| 人声分离 / 音视频分离 | 图二右键有；依赖 AI 或 FFmpeg 复杂滤镜，**视频节点工具栏多为 stub** |
| 智能去字幕、高清、解析 | 属 **视频节点** 生成链，非时间线基础项 |

## 四、建议的数据模型（P0 前置）

**已实现（迭代 19）** — `ffmpegConcat` 节点：

```ts
timelineClips: ComposeTimelineClip[];  // 主数据
inputs: string[];                      // 与 relPath 同步，兼容旧面板
```

导出：`render_timeline` 对每段按 `inSec`/`outSec` 裁切到 `.canvasflow/timeline-trim/` 再 concat。

## 五、UI 结构目标（对齐图二，非图一）

```
┌─────────────────────────────────────────────────────────┐
│ 剪辑                                    [导出▾]  [×]    │
├─────────────────────────────────────────────────────────┤
│                                                         │
│              预览（16:9，尽量吃满宽度）                    │
│                                                         │
├─────────────────────────────────────────────────────────┤
│ ↶ ↷ │ ✂ │ 🗑 │ … │      00:10 / 05:37  ▶      │ − ═ + ⊡ │
├─────────────────────────────────────────────────────────┤
│ ▶ 🔇 │ 00:00   00:30   01:00 …  │▌播放头                    │
│      │ [====胶片条片段====][片段2…]                         │
└─────────────────────────────────────────────────────────┘
```

**与当前实现差异（必须改的 UI 项）**

1. 时间线区域 **固定轨高**（如 72–88px），不要大块空轨。
2. 主工具栏 **固定 7 个槽位**：撤销、重做、分割、删除、（变速可 P1）、播放区、缩放区。
3. 片段上显示 **镜号/时长**（如 `10k 00:00:04:01`），不是仅文件名。
4. 预览 **默认隐藏** 原生 `controls`， transport 只在工具栏。
5. 顶栏标题读 `label`，新建默认 **「剪辑」**；提供「重命名为剪辑」迁移可选。

## 八、P0 验收场景（手动）

1. 连接 2 个视频节点 → 刷新 → 时间线出现 2 段，胶片可辨认内容。
2. 选中片段，拖播放头 → 预览画面跳转对应该时刻。
3. 在播放头 **分割** → 变为 3 段，顺序正确。
4. 裁掉片段左侧 2 秒 → 导出成片总时长减少约 2 秒。
5. 删除中间段 → 导出只含首尾内容。
6. **撤销** 一步可恢复删除/分割。
7. 顶栏导出成功，预览可切换「看成片」。

## 九、与画布其它入口的关系

| 入口 | 职责 |
|------|------|
| **视频节点 · 单段裁剪** | 单文件 in/out，替换节点成片 |
| **剪辑节点 · 时间线** | 多段顺序、裁切、分割、导出一条成片 |
| **视频节点 · 剪辑菜单 → 多段合成** | 打开/聚焦剪辑节点 |

避免两套裁切逻辑分叉：时间线裁切应复用同一 FFmpeg trim 参数与路径规范。

---

## 十、总结

- **图一**：布局骨架有了，但缺 **专业时间线数据模型、分割/裁切/撤销、可信缩略图**，工具栏与能力不对齐。  
- **基础剪辑** = 上表 **P0 的 14 项**；其中 **8–12、13–14** 是当前最大缺口。  
- 建议先落地 **迭代 19（数据模型 + trim 导出）**，再做 **20（编辑工具 + undo）**，最后 **21（视觉对齐图二）**。
