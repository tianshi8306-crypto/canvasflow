# 分镜脚本节点功能说明

> **用途**：产品 / 开发 / 验收共用的「脚本节点能做什么」真源说明。  
> **对齐**：[LibTV 使用指南 §1.2.6](LIBTV_GUIDE_ALIGNMENT.md)（交互范式，非云端 API 复刻）。  
> **实现对照**：蓝图见 [`脚本分析设计方案.md`](../iterations/脚本分析设计方案.md)；V1 边界见 [`脚本分析设计方案_V1落地版.md`](../iterations/脚本分析设计方案_V1落地版.md)；画布 Chrome 见 [`iteration-08-script-node-chrome.md`](../iterations/iteration-08-script-node-chrome.md)。

---

## 1. 定位（一句话）

**分镜脚本节点**是无限画布上的**叙事中枢**：把主题、剧本文本、（弱）参考素材约束，变成可编辑、可勾选、可绑定的结构化镜头表（`scriptBeats`），再产出分镜文案（`storyboardShots`），供图片 / 视频 / 合成节点按 `scriptBeatId` 消费。

```mermaid
flowchart LR
  A[创意输入] --> B[脚本节点]
  B --> C[scriptBeats 镜头表]
  C --> D[storyboardShots 分镜文案]
  D --> E[图片 / 视频 / 合成]
```

与 **文本节点** 的边界：文本节点承载自由正文与工作流；脚本节点承载**行级结构化镜头**与下游稳定主键，主编辑面是 **Inspector 工作台 + 全屏表**，不是节点内长文编辑器。

## 3. 能力四层模型（推荐对外表述）

### 3.1 输入层

| 方式 | 行为 | 状态 |
|------|------|------|
| **主题 / 解析要求** | 节点 `prompt`：剧情主题、风格与解析约束 | ✅ 底栏 Composer、Inspector、全屏可改 |
| **上游文本连线（剧本导入）** | 文本/LLM 节点连线 → 解析时 `prompt`=解析要求、上游正文=待解析剧本；壳/Inspector 显示字数并可定位 | ✅ script-11-2A |
| **同步到主题（旧）** | 不再作为推荐路径；长剧本应保留在文本节点 | ⚠️ 已改为「定位上游剧本」 |
| **参考视频** | 连入参考视频时，执行器收集路径写入解析上下文（无画面理解） | ⚠️ |
| **表内参考 / 角色图** | 全屏表选图、选视频 → 导入 `assets/` 填相对路径 | ✅ |
| **本地镜头模板** | 保存/应用/导入导出/预置包（工作台） | ⚠️ 非 Lib 模板市场 |
| **本机 .txt 选文件导入** | — | ❌ Out of Scope（用文本节点承载剧本） |
| **文档 Excel / 多模态反推** | — | ❌ V1 Out of Scope |

**说明**：R3 文档中的「四大输入来源」「六步核心生成闭环」是**产品愿景叙事**，不是已实现的分步向导。

### 3.2 结构化层（`scriptBeats`）

#### 三条不同的「生成」路径（必读，勿混用）

| 用户可见入口 | 技术路径 | 是否调 LLM |
|--------------|----------|------------|
| Inspector **「根据主题生成草案」** | `scriptDraftFromThemeAgentRuntime` | **否** — 本地生成 3 条模板镜头，便于立即编辑 |
| 底栏 **「生成镜头」**、顶栏 **「重新生成」**、全屏 **「重新生成」** | `scriptNodeDispatchAgentRuntime` → `runNodeSubgraph` → Rust `run_script_node` | **是** — 剧本解析 |
| **运行图 / DAG 执行** | 同上，由图调度触发 | **是** |

#### 后端解析产出（Rust `run_script_node`）

- 合并：解析要求 + 上游文本 +（可选）参考视频路径说明  
- 风格：`params` 风格配置或 `detect_style_from_text` 自动检测  
- 强约束 JSON：镜号递增、时长、景别/情绪/运镜枚举、`storyboardPrompt` / `videoMotionPrompt` 格式、角色五段描述等  
- 回写：`scriptBeats`、`scriptBeatSelection`（初始化）、`scriptShotCount` / `scriptTotalDurationSec` 等  

#### 单条镜头数据（`ScriptBeat` 摘要）

镜号、时长、画面描述、角色（含 `characters[]`）、参考、景别、运镜/场景标签、对白、分镜/视频提示词；可选 `timeIn` / `timeOut`（**无** Lib 级全片时间轴 UI 引擎）。

#### Provider

- 底栏 `ScriptModelPicker` 可写 `params.providerId`  
- 图执行解析以 **工程设置 + 节点 params** 为准；验收时需单独确认「底栏所选 = 解析所用模型」是否一致。

### 3.3 增强层（编辑 + 分镜文案）

#### 四处编辑 / 操作界面

| 界面 | 文件 / 入口 | 职责 |
|------|-------------|------|
| 画布壳 + Portal | `MinimalScriptNode` | 迷你表预览（**点击进全屏**）、顶栏解析/分镜/全屏/主题/下载、底栏主题与 AI 解析 |
| **Inspector 脚本工作台** | `ScriptNodeWorkbench` | **主编辑**：表/卡、批量、模板、分镜区 |
| **全屏 Overlay** | `ScriptNodeFullscreenOverlay` | 大表 + **创意视图** Tab；顶栏三键（分镜规则见下） |
| 节点最大化 | `NodeMaximizedOverlay` | 嵌工作台 + 分镜区（另一最大化入口） |

**产品决策**（iteration-08）：主题 → 底栏/Modal；镜头表 → 全屏为主；Inspector **保留**完整工作台；双击节点 → 画布 200% 居中，**不**直接开全屏。

#### 工作台编辑能力（摘要）

- 视图：表格 / 卡片  
- 行：增删、勾选、`scriptBeatSelection` 持久化  
- 批量：景别/运镜批量填写、收藏词、Enter 快捷应用  
- 镜号：排序、勾选重排、补零、仅保留勾选、删除勾选、**撤销批量镜号**、最近 3 条操作复用  
- 全屏：字段显隐/筛选及持久化、字段弹层键盘流、媒体入格  
- 模板：本地保存/应用/导入导出/预置包  
- **「进入分镜区」**：同步勾选并滚动到侧栏分镜文案区  

#### 分镜文案（`storyboardShots`）

- LLM 生成画面描述，按 `scriptBeatId` 与镜头表关联  
- Inspector：**「为勾选脚本生成分镜」** / **「为全部脚本生成分镜」**  
- 视图：网格/列表、全文弹层、本地改文案  
- 运维：缺失/空文案/失败分批重试、清理缓存（孤儿/去重）、健康统计  
- 本机分镜图：选图导入 `assets/` 关联预览（非云端批量出图）  

#### 「生成分镜」范围规则（验收必测）

共用 `resolveStoryboardBeatScope`（`src/lib/scriptStoryboardScope.ts`）：

| 规则 | 行为 |
|------|------|
| **默认（顶栏 / 全屏 / 侧栏「勾选」）** | 有有效勾选 → **仅勾选**；无勾选 → **全部** |
| **侧栏「为全部脚本生成分镜」** | 始终 **全部**（显式按钮，不走默认推断） |

| 位置 | 说明 |
|------|------|
| **画布顶栏** | 与全屏一致；按钮可显示 `生成分镜（N）` |
| **全屏顶栏** | 同上；完成后切 **创意视图** |
| **Inspector 分镜区** | 「勾选」= 默认规则；「全部」= 强制全部 |

#### 画布 Chrome（单选、有镜头时）

- 预览壳尺寸：与文本节点默认预览壳同宽同高（`scriptNodeChrome.ts`）  
- 壳内迷你表：最多 **3** 行，超出滚动提示全屏查看  
- 顶栏：**重新解析 | 生成分镜**（文案）\| **全屏表格 | 编辑主题 | 下载**（图标）  
- 壳内预览：**单击**打开 `ScriptNodeFullscreenOverlay`（双击节点仍为画布 200% 居中）  
- 底栏：主题、`@` / `/`、Provider、**AI 解析镜头**；可钉住 / 展开主题 Modal  

### 3.4 制片联动层

| 能力 | 说明 |
|------|------|
| **`scriptBeatId` 绑定** | 图片/视频 `params.scriptBeatId` + `shotNumber`；复制节点时 beatId 重映射 |
| **一键链路** | 分镜区为镜头创建 image/video 节点并写 prompt 草稿 |
| **批量视频** | 对已有关联视频节点批量提交（试点） |
| **合成导出** | `assessScriptComposeReadiness` 等评估后导出 |
| **Hermes 自动链** | 在 **已有 `storyboardShots`** 时，为分镜批量创建 image+video 并连线；**不是** parse 完就自动建完全部下游 |
| **导出 JSON** | 顶栏下载 / 全屏导出（`scriptNodeExport`） |
| **文本反向同步** | 文本节点可从上游脚本同步正文（`textScriptSync`） |

## 4. 主数据契约（工程 JSON）

| 字段 | 含义 |
|------|------|
| `prompt` | 剧情主题 / 解析要求 |
| `scriptBeats[]` | 结构化镜头表（真源） |
| `scriptBeatSelection[]` | 勾选镜头 id |
| `storyboardShots[]` | 分镜文案，键为 `scriptBeatId` |
| `scriptShotCount` / `scriptTotalDurationSec` | 统计（执行器可回填） |
| `params.providerId` / `model` | 可选模型覆盖 |
| `status` | 节点运行状态（生成中进度等） |

下游绑定：**禁止**仅靠镜号字符串匹配，以 `scriptBeatId` 为准。

## 6. 验收速查（手工）

1. **草案**：点「根据主题生成草案」→ 应立即出现 3 条模板镜头（无需等 LLM）。  
2. **解析**：底栏「AI 解析镜头」或顶栏「重新解析」→ 应走 DAG；busy 与 `isGraphRunning` / 节点 `status` 一致；0 条时有运行日志引导。  
3. **分镜失败**：全屏创意视图「定位镜头」可高亮脚本表对应行。  
4. **分镜范围**：分别在画布顶栏、全屏（有/无勾选）、Inspector 分镜区验证「勾选 vs 全部」。  
5. **持久化**：改表 → 保存工程 → 重开，`scriptBeats` / `storyboardShots` / 勾选仍在。  
6. **下游**：从脚本连 image/video，检查 `params.scriptBeatId`；粘贴含脚本子图时 beatId 已重映射。  

更细步骤见 [`R3_小白验收操作指南_2026-04.md`](../iterations/R3_小白验收操作指南_2026-04.md)。
