# 整理画布功能设计

## 交互流程

1. 用户点击「整理画布」按钮
2. 系统自动保存当前布局到历史记录
3. 执行整理算法（适配所有节点到视窗）
4. 显示内联确认按钮：「还原」「保留」
5. 用户点击「还原」→「从历史记录恢复节点原位置」
6. 用户点击「保留」→「完成，确认菜单消失」

## 确认按钮组设计

- 直接显示在「整理画布」按钮下方
- 两个小按钮横向排列
- 「还原」：带撤销图标
- 「保留」：带勾选图标
- 菜单保持显示，不自动消失

## 状态管理

- 整理前自动保存完整节点状态到 `historyStore`
- 撤销操作从历史记录恢复
- 保留后不清理该条历史（用户可在历史面板撤销）

## 快捷键

- 不支持键盘快捷键，只用鼠标点击

## 实现文件

- `src/components/canvas/CanvasFlowChrome.tsx` - UI 组件
- `src/store/historyStore.ts` - 历史记录管理（如果不存在需要创建）
