# R2 小白验收操作指南（不会编程版）

> 目标：你只需要按步骤点击界面，最后告诉我“通过/不通过”即可。  
> 全程不需要写代码、不需要懂技术名词。

---

## 0. 先做准备（2 分钟）

1. 打开 Cursor 里的终端。
2. 运行：`npm run tauri dev`
3. 等桌面应用窗口弹出来（看到画布就行）。

如果这一步报错，不用处理，直接把报错截图发我。

---

## 1. 创建 5 种节点（3 分钟）

在画布上添加 5 个节点：文本、图片、视频、音频、脚本。

你只看这 3 件事：

- 文本节点标题下是否有提示：`尚无正文；双击卡片或选中后编辑`
- 脚本节点标题下是否有提示：`尚无脚本镜头；可写梗概或使用生成`
- 图片/视频/音频节点标题下是否有提示：`未绑定素材；可上传或拖入工程`

结果记录：

- 都看到了 = 通过
- 任意一个没看到 = 截图给我

---

## 2. 复制/粘贴/删除（4 分钟）

### 2.1 文本节点

1. 双击文本节点，输入一句话（例如：`这是一次测试`）。
2. 选中文本节点，按 `Ctrl + C`，再按 `Ctrl + V`。
3. 看看是否出现一个复制出来的新节点（位置会稍微错开）。

### 2.2 脚本节点

1. 选中脚本节点，`Ctrl + C` 后 `Ctrl + V`。
2. 看是否能正常出现副本，不报错不卡住。

### 2.3 删除

1. 选中任意一个复制出来的节点。
2. 按 `Delete`。
3. 看节点是否消失，且关联线也一起消失。

结果记录：

- 全部正常 = 通过
- 有异常 = 录屏 10 秒给我

---

## 3. 拖拽导入素材（4 分钟）

1. 从资源管理器拖 3-5 个文件到画布（图片/视频/音频都可混合）。
2. 观察：
  - 会出现“导入中”的提示
  - 导入后会生成多个节点
  - 这些节点不会全部堆在一起（会错位排开）
3. 再试一次：混入 1 个不支持文件（比如 `.txt`）。
4. 观察：
  - 支持的文件仍导入成功
  - 有提示“跳过了不支持文件”

结果记录：

- 都正常 = 通过
- 不正常 = 截图状态栏 + 画布

---

## 4. 保存并重开（3 分钟）

1. 先保存工程。
2. 关闭应用窗口，再重新打开同一个工程。
3. 看 3 件事：
  - 节点还在
  - 连线还在
  - 节点里刚才填的内容还在（如文本内容、素材路径）

结果记录：

- 都还在 = 通过
- 丢了任何一个 = 截图给我
