# 第 2 轮执行单：画布与五节点体验重构（低复杂度）

> 合并验收时可对照：`R2_QA_CHECKLIST.md`（同目录）。

## 本轮目标

将“能用”升级为“可创作”：画布操作节奏顺滑、五类节点交互一致、拖拽导入可预期。

## 范围锁定（只做这些）

1. 画布体验规范
  - 节点卡片结构统一（标题、内容区、状态提示区）。
  - 画布操作反馈统一（选中、拖拽、连线、删除）。
2. 五类节点体验一致化
  - 文本/图片/视频/音频/脚本节点均具备最小“编辑 + 预览 + 路径/内容可见”闭环。
3. 拖拽导入体验补齐
  - 导入中、导入成功、导入失败反馈可见。
  - 批量导入时节点自动错位排布，避免堆叠。

## 本轮不做（明确排除）

- 不做脚本批量转分镜。
- 不做高级画布工具（复杂右键工具箱、小地图增强）。
- 不做节点执行调度与容错策略增强。

## 交付清单（最多 4 项）

- 五类基础节点交互逻辑一致，且可直接用于创作输入。
- 节点基础操作（复制/粘贴/删除/连线）行为稳定且反馈清晰。
- 拖拽导入支持批量素材，且导入结果可读可回看。
- 重开工程后节点内容与连线状态可恢复。

## 手工验收步骤（可视化，零代码）

1. 新建工程后依次创建文本、图片、视频、音频、脚本节点并填写最小内容。
2. 对每类节点执行选中、复制、粘贴、删除与连线操作，确认反馈一致。
3. 拖入 3-5 个本地素材（混合图片/视频/音频），确认导入成功提示和节点自动排布。
4. 故意拖入一个不支持或异常文件，确认失败提示可读且不影响其他素材导入。
5. 保存并重启后打开工程，确认节点内容、素材路径、连线全部恢复。

## UI/UX

> 详见 [UI 迭代指南](../design/UI_ITERATION_GUIDE.md)。

- **关键界面**：画布、节点卡、侧栏 Inspector、右键/添加面板、拖拽覆盖层、顶栏状态。
- **关键状态**：选中、拖拽、连线、导入中/成功/失败；禁用控件配合 `title`。
- **键盘与焦点**：在节点标题/正文输入时避免误触全局删除等快捷键（见验收与 `global.css` 约定）。

## 风险与回退

- 风险：
  - 节点视觉统一后，局部交互被意外破坏。
  - 批量拖拽导入在部分文件格式下失败率偏高。
- 回退触发条件：
  - 创作主路径（创建节点->编辑->连线->保存）任一步骤明显不稳定。
- 回退策略：
  - 回退到第 1 轮稳定版本，仅保留 R2 文档与体验问题清单。
- 回退后保留：
  - 失败素材包、操作录屏、报错截图、复现步骤。
