# 迭代执行模板（低复杂度）

> 目标：每轮只做一件大事，改动小、验收清楚、问题可回退。

## 1) 本轮目标（一句话）

- 示例：完成“工程创建/打开/保存”的最小闭环。

## 2) 变更范围（最多 3 个模块）

- 模块 A：
- 模块 B：
- 模块 C（可选）：

## 3) 功能清单（2-4 项）

- 功能点 1
- 功能点 2
- 功能点 3（可选）
- 功能点 4（可选）

## 4) 非目标（本轮不做）

- 不做项 1
- 不做项 2

## 5) 验收步骤（3-5 步，纯手工可验证）

1. 步骤 1
2. 步骤 2
3. 步骤 3
4. 步骤 4（可选）
5. 步骤 5（可选）

## 6) UI/UX（按轮填写，可简写）

> 详见 [UI 迭代指南](../design/UI_ITERATION_GUIDE.md)。若本轮**无界面变更**，此处写一行：**「本轮无 UI 变更」**。

- **关键界面**（画布 / 侧栏 / 顶栏 / 弹层等）：
- **关键状态**（空 / 加载 / 成功 / 失败 / 禁用及 `title`）：
- **键盘与焦点**（与全局快捷键、输入框冲突的规避）：
- **本轮 UI 非目标**（可选，如不改品牌色）：

## 7) 风险与回退

- 主要风险：
- 触发条件（出现什么现象就判定失败）：
- 回退动作（回到上轮稳定版本）：
- 回退后保留信息（日志/截图/失败输入）：

## 8) 完成定义（DoD）

- 功能点全部通过手工验收
- 关键路径无阻塞错误
- 已满足本节「UI/UX」要求（或已声明「本轮无 UI 变更」）
- 相关文档已更新
