# 第 3 轮执行单：脚本工作台 v1（低复杂度）

## 本轮目标

建立脚本工作台的最小闭环：脚本可生成、可编辑、可切视图、可作为后续分镜生成输入。

## 范围锁定（只做这些）

1. 脚本生成入口
   - 支持“主题/剧情描述 -> 脚本草案”最小生成流程。
2. 脚本编辑与展示
   - 支持基础字段编辑（场次、镜头描述、时长建议）。
   - 支持表格视图与卡片视图切换。
3. 脚本批量操作最小集
   - 支持单条/多条勾选与批量发送到下一步生成入口（仅准备，不执行分镜生产）。

## 本轮不做（明确排除）

- 不做分镜图批量生成执行。
- 不做视频生成与导出链路。
- 不做 Provider 高级编排（该能力后移至 R7）。

## 交付清单（最多 4 项）

- [x] 可从脚本节点触发最小脚本生成并落地在工作台中（「根据主题生成草案」+ `scriptBeats` 存工程）。
- [x] 脚本字段可编辑，且保存后可恢复（侧栏工作台 + `canvasflow.json`）。
- [x] 表格/卡片两种视图可切换且内容一致。
- [x] 可勾选脚本条目并批量发送到后续入口（占位动作 + `scriptBeatSelection` 持久化以便重开恢复勾选）。

## 手工验收步骤（可视化，零代码）

1. 新建脚本节点，输入剧情主题并触发脚本生成。
2. 在脚本工作台修改 2-3 条脚本字段并保存工程。
3. 切换“表格视图/卡片视图”，确认内容无丢失。
4. 勾选多条脚本执行批量发送动作，确认反馈可见。
5. 重启后重新打开工程，确认脚本内容与勾选状态可恢复。

## UI/UX

> 详见 [UI 迭代指南](../design/UI_ITERATION_GUIDE.md)。

- **关键界面**：侧栏脚本工作台（表格/卡片）、画布脚本节点摘要、批量发送占位入口。
- **关键状态**：生成中/空列表/编辑保存失败；勾选与持久化恢复一致。
- **键盘与焦点**：表格/卡片内编辑与全局快捷键不冲突。

## 风险与回退

- 风险：
  - 视图切换导致编辑状态丢失。
  - 批量操作入口可见但状态同步不一致。
- 回退触发条件：
  - 脚本生成或编辑任一关键步骤不稳定，无法支撑后续分镜阶段。
- 回退策略：
  - 回退到 R2 稳定版本，保留脚本工作台 UI 草案与失败样本。
- 回退后保留：
  - 失败脚本样本、交互录屏、错误日志与复现步骤。
