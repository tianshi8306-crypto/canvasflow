# 第 6 轮执行单：脚本全屏表格（对齐 Lib 1.2.6）

## 本轮目标

在脚本工作台保留侧栏「表格 / 卡片」的前提下，增加 **全屏表格** 模式：大视口编辑与侧栏同一 `scriptBeats` 真源，支持 Esc / 点击背景关闭。

## 范围锁定

1. 抽取共用组件 `ScriptBeatsEditorTable`（内嵌表 + 全屏表）。
2. 工具栏「全屏表格」入口；全屏顶栏复用：全选、清除勾选、生成草案、添加镜头、占位发送、关闭。
3. 全屏描述列为更大 `textarea`（行数与最小高度）。
4. 全屏字段工具增强：字段显隐、筛选、键盘快捷流、状态持久化。
5. 媒体入格增强：角色图/参考视频支持本地选择后导入 `assets` 并回填字段。

## 本轮不做

- 改变卡片视图布局逻辑。

## Lib 指南对齐

- 指南 **1.2.6 脚本节点** 中的「全屏视图 / 表格编辑」体验子集。

## 手工验收

1. 选中脚本节点，有若干 `scriptBeats`。
2. 点击「全屏表格」，表格铺满中央视区，编辑一格后关闭，侧栏表格数据一致。
3. 全屏内「根据主题生成草案」后关闭，侧栏同步。
4. 按 **Esc** 或点击暗色背景关闭全屏。
5. 无条目时打开全屏，可见空状态提示，可用顶栏「添加一条镜头」。

## 增量能力（2026-04-24）

- **字段显隐可用性**
  - 字段弹层支持：`全部显示` / `恢复默认` / `全部隐藏（保留镜号）`。
  - 提供可见列计数：`已显示 X / Y 列`。
  - 最少保留 1 列可见，防止误操作导致“空表”。
- **字段筛选与持久化**
  - 行筛选关键词持久化（`localStorage`），重开后可恢复。
  - 字段显隐配置持久化（`localStorage`），重开后可恢复。
  - 字段搜索词会话级保留（`sessionStorage`），同一会话内关闭再开仍保留。
- **键盘效率流**
  - 筛选框：`Esc` 清空并关闭筛选弹层。
  - 字段搜索框：`Esc` 清空（不关闭弹层）；`Tab/Shift+Tab` 与 `↑/↓` 切换匹配项；`Enter` 切换当前项显隐。
  - 字段弹层底部有快捷键提示文案，降低学习成本。
- **媒体入格**
  - 全屏表格中 `角色图`（选图）与 `参考`（选视频）支持本地选择 -> 导入 `assets` -> 自动写回相对路径。
  - 浏览器预览环境给出限制提示；桌面端可正常执行。

## 增量验收步骤（建议补测）

1. 打开全屏字段弹层，依次执行「全部显示 / 恢复默认 / 全部隐藏」，确认可见列计数正确且始终至少保留 1 列。
2. 在字段搜索框输入关键词，使用 `Tab` 或 `↑/↓` 移动候选，按 `Enter` 切换显隐；按 `Esc` 仅清空搜索词。
3. 在行筛选弹层输入关键词并关闭全屏，再次打开，确认筛选词仍在且结果一致；字段显隐配置同理。
4. 在 `角色图` 与 `参考` 列点击选择按钮导入媒体，确认文件进入 `assets` 且单元格自动填入相对路径。

## UI/UX

- **关键界面**：全屏遮罩 + 顶栏操作区 + 大表。
- **键盘**：Esc 关闭与弹层优先级关闭；字段与筛选弹层支持键盘高频操作。
- **z-index**：全屏层高于分镜弹层（约 95），避免被遮挡。

## 风险与回退

- 风险：小屏顶栏按钮换行过多。
- 回退：移除 `ScriptNodeWorkbench` 全屏相关状态与 `ScriptBeatsEditorTable` 文件，恢复内联表结构。
