# 脚本分析设计方案 V1 落地版（开发执行单）

**版本**：V1.0  
**定位**：本期可交付、可测试、可验收的最小闭环  
**对应关系**：承接《脚本分析设计方案（蓝图版）》，只实现当前仓库可稳定落地部分

---

## 1. 本期目标（必须达成）

打通脚本节点端到端主链路：

1. 输入进入脚本节点（手动输入 / 上游文本连线）
2. 调用设置页可配置的 Provider 与 API Key
3. 执行脚本解析，提取结构化分镜字段
4. 回写标准化 `scriptBeats`
5. 下游图片/视频节点可按 `scriptBeatId` 自动绑定消费

---

## 2. 本期范围（In Scope）

### 2.1 输入来源（本期）
- 上游文本节点连线输入（DAG 合并）
- 脚本节点手动输入（解析要求）

### 2.2 解析输出（本期）
- `scriptBeats` 标准字段回写（含镜号、时长、描述、角色、景别、运镜、提示词等）
- `scriptBeatSelection` 自动初始化
- `scriptTotalDurationSec` / `scriptShotCount` 统计字段回写

### 2.3 模型与配置（本期）
- 文本节点 / 脚本节点可选择 Provider（来源于设置页 `providers`）
- 节点参数支持覆盖 `providerId` / `model`
- 未覆盖时走设置页默认优先级 Provider

### 2.4 联动与可观测（本期）
- `script_parse_request / attempt / retry / failed / response` 事件完整
- 下游 image/video/audio 节点按脚本选择自动补齐 `scriptBeatId + shotNumber`

---

## 3. 本期不做（Out of Scope）

- 参考视频/图集反向解析分镜（多模态）
- Excel 导入（`xls/xlsx`）
- 模板市场 / 插件市场
- 全局加密体系（AES 全链路）与高级容灾工具
- “相同输入逐字一致”绝对确定性承诺（本期仅保证结构确定性）

---

## 4. 结构确定性定义（本期验收口径）

本期“确定性”定义为：

- 字段结构确定：输出 JSON Schema 稳定、必填字段完整
- 业务规则确定：镜号连续、时序可计算、枚举字段落在允许集合
- 错误行为确定：失败路径有明确错误与重试策略

不要求模型生成文本逐字一致。

---

## 5. 关键数据契约（V1）

### 5.1 ScriptBeat 主键与绑定
- `id`：分镜唯一主键
- `shotId`：与 `id` 对齐，供跨节点稳定绑定
- 下游绑定统一使用 `scriptBeatId`，禁止仅靠镜号文本匹配

### 5.2 必达字段（最小集）
- `shotNumber`
- `durationHint`
- `description`
- `shotSize`
- `sceneTags`（运镜）
- `storyboardPrompt`
- `videoMotionPrompt`
- `characters[]`（允许为空）

---

## 6. 执行流程（V1）

1. DAG 触发脚本节点执行
2. 收集输入（上游文本 + 当前 prompt）
3. 组装系统提示词与用户提示词
4. 调用 Provider（支持节点级 `providerId/model` 覆盖）
5. 解析 JSON，失败按策略重试（最多 3 次）
6. 归一化为 `scriptBeats`
7. 回写节点 patch 与运行事件

---

## 7. 验收标准（V1）

### 7.1 功能验收
- 上游文本 + 脚本节点输入要求可触发解析并生成 `scriptBeats`
- 切换 Provider 后，脚本解析调用跟随变化
- 解析失败会显示可读错误，不会静默失败

### 7.2 数据验收
- `scriptBeats` 可持久化，重开工程可恢复
- `scriptBeatId` 绑定链路在下游节点有效

### 7.3 回归验收
- `npm run typecheck` 通过
- `npm run test` 通过
- 小白验收可按 `R3_小白验收操作指南_2026-04.md` 走通
