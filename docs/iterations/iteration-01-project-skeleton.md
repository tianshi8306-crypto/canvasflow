# 第 1 轮执行单：项目骨架稳定（低复杂度）

## 本轮目标
确保客户端稳定启动，工程可创建/打开/保存，并给出清晰状态反馈。

## 范围锁定（只做这些）
1. 工程目录规范
   - 自动初始化 `canvasflow.json`、`assets/`、`.canvasflow/`。
2. 工程生命周期操作
   - 新建工程、打开工程、保存工程。
3. 基础状态反馈
   - 顶部或状态栏明确显示：当前工程路径、保存状态、最近一次操作结果。

## 本轮不做（明确排除）
- 不做节点执行增强。
- 不做多 Provider 调度策略优化。
- 不做 Agent 自动编排。
- 不做时间线高级编辑能力。

## 交付清单（最多 4 项）
- [ ] 工程初始化结构一致且可重复打开。
- [ ] 新建/打开/保存操作均可正常使用。
- [ ] 保存失败时有明确错误提示。
- [ ] 关闭重开后工程状态可恢复。

## 手工验收步骤（可视化，零代码）
1. 启动客户端，创建一个全新工程目录。
2. 确认目录中存在 `canvasflow.json`、`assets/`、`.canvasflow/`。
3. 在画布执行一个可见改动（如新建节点或调整视口）并点击保存。
4. 关闭客户端后重新打开同一工程，确认改动仍在。
5. 观察状态反馈是否正确显示“已保存/保存失败/当前工程路径”。

## UI/UX

> 详见 [UI 迭代指南](../design/UI_ITERATION_GUIDE.md)。

- **关键界面**：主壳顶栏/状态区（工程路径、保存态）。
- **关键状态**：保存中、已保存、保存失败；打开工程失败时有可读错误。
- **键盘与焦点**：本轮以工程操作为主，无新增全局快捷键冲突项。

## 风险与回退
- 风险：
  - 工程文件写入失败导致保存状态与真实落盘不一致。
  - 旧工程兼容性问题导致打开失败。
- 回退触发条件：
  - 无法稳定打开已有工程，或保存后重开数据丢失。
- 回退策略：
  - 回退到上一稳定迭代版本，仅保留文档与验收记录。
- 回退后保留：
  - 错误日志、失败工程目录样本、复现步骤。
