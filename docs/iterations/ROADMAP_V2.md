# AI短剧平台路线图（V3，创作链路驱动）

> 文件名保留 `ROADMAP_V2.md`；正文为 V3（创作链路驱动）版本。

产品层面对齐（无限画布 + 五类节点 + 脚本→分镜→视频等）见 [LibTV 指南对齐说明](../product/LIBTV_GUIDE_ALIGNMENT.md)，避免迭代与「本地 LibTV 式」目标脱节。

## 迭代原则

- 每轮只做 1 个核心目标。
- 每轮最多 3 个模块、2-4 个功能点。
- 每轮必须有 3-5 步手工验收。
- 每轮必须定义失败触发条件与回退策略。
- 路线必须围绕“从主题到成片”的连续创作链路推进。

## 方案分层（四层）

### 创作体验层

- 画布交互、节点体验、属性面板一致性。
- **设计与体验（子维度，与上项同一层交付）**
  - **信息架构**：主画布、侧栏、弹层/面板的职责与入口，避免同一任务多路径冲突。
  - **视觉与组件**：以 `[src/styles/global.css](../../src/styles/global.css)` 中语义变量（如 `--bg`、`--panel`、`--accent`）为真源；节点卡、工作台表格、顶栏状态等密度与层级统一。
  - **状态与反馈**：加载/空/错/成功态可读；禁用控件配合 `title` 说明原因；键盘焦点与关键路径不误伤（例如在输入框内不按退格误删画布）。

详细写法与按轮 UI 强度见 [UI 迭代指南](../design/UI_ITERATION_GUIDE.md)。

### 生产链路层

- 主题输入 -> 脚本 -> 分镜 -> 视频 -> 时间线 -> 导出。

### 能力编排层

- Provider 管理、模型路由、调用日志与容错。

### 资产与质量层

- 资产沉淀、历史复用、质量门禁与发布验收。

## 9 轮路线与边界

### R1 基础稳定

- 核心目标：工程创建/打开/保存 + 状态反馈稳定。
- **UI 焦点**：顶栏工程路径、保存态与错误提示清晰；主壳布局稳定。
- 边界：不做节点体验重构，不做复杂调度。

### R2 画布与五节点体验重构

- 核心目标：五类基础节点达到“可编辑 + 可预览 + 可复用”的一致体验。
- **UI 焦点**：节点卡（标题/正文/预览区）与侧栏 Inspector 对齐；画布右键/添加面板、拖拽覆盖层与顶栏状态文案层级一致。
- 边界：不做分镜批量生成，不做视频导出链路。

### R3 脚本工作台 v1

- 核心目标：脚本生成、编辑、视图切换和基础批量操作。
- **UI 焦点**：侧栏脚本工作台表格/卡片布局、勾选与状态栏占位反馈；与画布脚本节点摘要一致。
- 边界：不做分镜图批量生产，不做模型路由优化。

### R4 分镜生成与批量操作 v1

- 核心目标：脚本到分镜的单条/批量生成和结果回看。
- **UI 焦点**：分镜结果以网格或列表清晰呈现；单条回看与大图预览入口明确，批量操作与进度状态不遮挡主列表。
- 边界：不做视频模型高级参数编排。

### R5 视频生成与模型参数面板 v1

- 核心目标：分镜到视频的可控生成与参数面板。
- **UI 焦点**：参数分组（基础/高级/危险操作）与默认值展示；生成中与失败态在侧栏或独立面板中有固定位置，避免与画布选中态混淆。
- 边界：不做多厂商自动切换与成本优化。

### R6 时间线合成与导出 v1

- 核心目标：基础时间线编辑与 mp4 成片导出闭环。
- **UI 焦点**：时间线控件可读性与缩放/拖拽反馈；导出进度、完成与失败提示不阻塞继续编辑。
- 边界：不做多轨高级特效和复杂剪辑。

### R7 API 编排增强

- 核心目标：Provider 路由、容错、费用可视化与调用可观测。
- **UI 焦点**：日志/费用/路由等信息密度与可读性；错误与重试状态在面板中有稳定展示位。
- 边界：不做 Agent 全自动编排。

### R8 资产库、历史记录、模板复用

- 核心目标：素材/脚本/工作流资产化与历史复用效率提升。
- **UI 焦点**：资产库浏览、筛选与预览的信息架构；模板入口与「应用到画布」反馈一致。
- 边界：不做社区发布和多人协作。

### R9 Agent 自动编排与稳定化发布

- 核心目标：一键编排、方案级验收、发布前稳定化。
- **UI 焦点**：编排向导步骤与发布前检查结果的可扫读布局；关键阻塞项突出。
- 边界：合规/人像授权等高依赖能力仅做预留或后置扩展。
