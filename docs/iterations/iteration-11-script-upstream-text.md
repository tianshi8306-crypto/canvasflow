# 第 11 轮执行单：上游文本节点剧本导入（script-11-2A）

> **状态**：已完成（2026-05-21）  
> **依据**：[脚本节点开发顺序.md](../product/脚本节点开发顺序.md) 阶段 2-A（**修订**：非 .txt 文件导入）  
> **功能真源**：[脚本节点功能说明.md](../product/脚本节点功能说明.md) §3.1

## 1) 本轮目标（一句话）

**剧本导入 = 文本节点（或 LLM）连线到脚本节点**；解析时 Rust 将底栏 `prompt` 作【解析要求】、上游正文作【待解析剧本文本】，UI 可发现、可定位。

## 2) 变更范围

- `src/lib/scriptUpstreamText.ts` + 单测
- `ScriptUpstreamTextBanner`（画布 float + Inspector 面板）
- `MinimalScriptNode`、`ScriptComposerPanel` 提示
- 文档修订（取消 txt 选文件方案）

## 3) 功能清单

- [x] 列出已连接上游文本源（标签、字数、预览）
- [x] 画布：有连线即显示「上游剧本 · N字」，点击定位文本节点
- [x] Inspector：剧本导入说明 + 源列表 + 定位
- [x] 底栏 Zone A：有上游时显示字数提示
- [x] 不再以「从文本同步」覆盖整段 prompt 作为唯一导入路径（保留 DAG 双字段语义）

## 4) 非目标

- `.txt` / 文件选择对话框导入
- 自动解析（仍由用户点「AI 解析镜头」）
- Word / Excel

## 5) 验收步骤

1. 文本节点写入 ≥1k 字剧本 → 连线到脚本节点（`text` → `script` 端口）。
2. 选中脚本节点 → 壳上显示「上游剧本 · X字」→ 点击定位到文本节点。
3. Inspector 面板列出源节点与字数，文案说明双字段解析规则。
4. 底栏只写简短解析要求 →「AI 解析镜头」→ `scriptBeats` 合理；运行日志可见 `hasUpstreamText`。
5. 禁用连线 → 提示「上游已禁用」，解析不读该文本。

## 6) UI/UX

- **float**：紧凑字数徽章，hover 显示完整说明。
- **inline**：teal 边框信息区，每条源可「定位」。

## 7) 风险与回退

- **风险**：用户仍习惯「同步到主题」旧流程。
- **回退**：恢复 `ScriptNodeUpstreamTextFloat` 旧「从文本同步」实现。
