# 第 4 轮执行单：分镜生成与批量操作 v1（低复杂度）

## 本轮目标

在脚本条目已就绪的前提下，完成**脚本到分镜文案**的单次/批量 LLM 生成与侧栏内回看（网格/列表），结果持久化到工程；不执行文生图或视频模型调用。

## 范围锁定（只做这些）

1. **分镜文案生成**
   - 对脚本工作台中**已勾选**或**全部**脚本条目，调用已配置的 OpenAI 兼容 Provider，生成每条对应的画面描述（`visualPrompt` 等）。
2. **结果展示与回看**
   - 侧栏提供**网格**与**列表**两种视图；支持弹层查看/编辑全文（画面描述、构图补充、负面提示）。
3. **持久化**
   - 分镜结果写入脚本节点 `storyboardShots`（与 `scriptBeatId` 关联），随 `canvasflow.json` 保存与恢复。

## 本轮不做（明确排除）

- 不做分镜图/视频的实际生成（文生图、图生视频属后续轮次）。
- 不做视频模型高级参数编排（属 R5 边界）。
- 不做 Provider 路由、费用统计等编排增强（属 R7）。

## 交付清单（最多 4 项）

- [x] Tauri 暴露 `llm_complete_text`（system + user），与工作流内 LLM 调用共享 OpenAI 兼容实现。
- [x] 脚本节点侧栏「分镜文案」区：为勾选/全部生成、网格与列表、全文弹层与本地编辑写回。
- [x] `storyboardShots` 类型与工程 JSON 序列化一致；重开工程可恢复。
- [x] 生成中按钮禁用与顶栏状态文案反馈。

## 手工验收步骤（可视化，零代码）

1. 打开工程，选中脚本节点，在脚本工作台至少有 2 条条目（可「根据主题生成草案」）。
2. 勾选其中 1 条，点击「为勾选脚本生成分镜」，确认状态栏提示完成且网格中对应卡片出现文案摘要。
3. 切换到「列表」，点击「全文」，编辑 `visualPrompt` 后失焦关闭，保存工程并重开，确认修改仍在。
4. 点击「为全部脚本生成分镜」，确认未勾选时也能覆盖/补充各条分镜文案（以模型返回为准）。
5. 在设置中关闭 API Key 或断网时再试生成，确认错误提示可读、界面不卡死。

## UI/UX

> 详见 [UI 迭代指南](../design/UI_ITERATION_GUIDE.md)。

- **关键界面**：脚本节点侧栏分镜区（网格/列表、工具栏、全文弹层）。
- **关键状态**：生成中禁用按钮；解析失败有状态说明；空脚本时引导先添加条目。
- **键盘与焦点**：弹层内 `TEXTAREA` 与全局快捷键策略一致；关闭弹层后焦点回到侧栏。
- **本轮 UI 非目标**：不重绘脚本表格样式、不改节点卡画布外观。

## 风险与回退

- **风险**：模型返回非 JSON 或 `scriptBeatId` 错位导致解析失败；长剧本 token 超限。
- **回退触发条件**：生成反复失败或错误写坏工程 JSON，无法打开工程。
- **回退策略**：回退到 R3 稳定提交；从备份 `canvasflow.json` 恢复 `storyboardShots` 字段（可整段删除该字段）。
- **回退后保留**：失败时的模型原始返回文本（可贴 issue）、Provider 配置截图、条目数量与主题摘要。
