## 一、方案核心设计原则
本模块所有设计严格遵循以下6条不可突破的核心原则，适配本地化单机软件的运行特性与业务需求：
1.  **离线优先原则**：核心解析能力100%可在无网络环境下运行，完整适配本地开源大模型，不强制依赖云端API，保证单机软件的核心功能离线可用
2.  **确定性输出原则**：通过固定规则模板、强格式约束、可复现的解析逻辑，杜绝大模型幻觉，保证相同输入可得到完全一致的输出结果，适配商业量产的稳定性需求
3.  **强校验与容错原则**：全流程多层级格式与内容校验，内置分级容错与重试机制，避免单次解析失败导致全流程中断，同时保证输出数据100%符合节点数据规范
4.  **低耦合高内聚原则**：模块内部职责闭环，与外部模块仅通过标准化数据接口交互，不依赖外部模块的内部实现，可独立开发、测试、迭代
5.  **数据本地化原则**：所有解析过程、用户数据、模板配置、生成结果，100%存储在用户本地磁盘，不上传任何数据到云端，保证用户剧本、项目数据的隐私安全
6.  **可配置可扩展原则**：所有解析规则、模板、格式约束均支持用户可视化配置，无需修改代码即可适配短剧/电影/广告/漫剧等不同创作场景，支持后续功能扩展

## 二、模块定位与边界定义
### 2.1 模块在软件架构中的位置
本模块是**脚本节点的核心执行内核**，隶属于软件DAG工作流引擎的算子层，向上对接脚本节点的双模块架构（上方资产容器模块+下方控制面板模块），向下对接本地大模型推理层，同时为下游图片/视频/TTS/合成节点提供标准化输出数据。
```
软件整体架构层级关系：
├─ 应用层：脚本节点双模块（UI交互/资产存储）
│  └─ 核心算子层：本方案「脚本解析模块」（核心执行内核）
│     ├─ 依赖1：DAG工作流引擎（调度/状态管控）
│     ├─ 依赖2：本地大模型推理层（离线语义解析）
│     └─ 依赖3：本地项目持久化层（数据存储/读取）
└─ 下游节点：图片/视频/TTS/合成节点（消费本模块输出数据）
```

### 2.2 模块职责边界
#### 【本模块核心负责范围】
1.  脚本节点所有输入内容的标准化预处理与清洗
2.  剧本/创意文本的语义解析，完成分镜结构化拆分
3.  全局角色体系的构建与IP特征锁定
4.  基于固定规则的分镜画面/视频运动提示词自动生成
5.  分镜全局时序坐标系的自动计算
6.  解析结果的格式校验、容错处理与异常修复
7.  解析结果的本地持久化存储与版本管理
8.  解析规则、模板、参数的可视化配置管理

#### 【本模块不负责范围】
1.  不负责DAG工作流的拓扑调度与节点状态管控（由DAG引擎负责）
2.  不负责AI画面/视频的生成执行（由下游生成节点负责）
3.  不负责大模型的本地部署与推理环境管理（由本地推理层负责）
4.  不负责前端UI的渲染与用户交互事件处理（由脚本节点UI层负责）
5.  不负责视频剪辑、音频合成、成片导出等后期功能（由合成节点负责）

## 三、模块整体架构设计
本模块采用**五层流水线架构**，数据单向流转，每层职责单一闭环，层与层之间通过标准化数据结构交互，便于分模块开发、测试与迭代。

### 3.1 整体架构分层
```
脚本解析模块流水线架构
├─ 第一层：输入预处理层
│  核心职责：多源输入的统一清洗、格式标准化、无效内容过滤
├─ 第二层：核心解析引擎层
│  核心职责：语义理解、分镜拆分、角色提取、剧情结构化拆解
├─ 第三层：Prompt工程规则引擎层
│  核心职责：固定格式提示词生成、全局变量替换、风格模板适配
├─ 第四层：输出校验与格式化层
│  核心职责：全字段格式校验、容错修复、时序计算、结果标准化
└─ 第五层：本地持久化与交互层
    核心职责：解析结果本地存储、版本快照、与上下游模块的数据交互
```

### 3.2 核心数据流转流程
```
1. 触发解析 → 2.多源输入归一化清洗 → 3.剧本语义解析与分镜拆分 → 4.全局角色体系构建
→ 5.固定格式提示词自动生成 → 6.全局时序坐标系计算 → 7.全量数据格式校验与容错
→ 8.结果持久化存储 → 9.标准化数据输出到上下游模块
```

## 四、输入与输出规范
### 4.1 输入规范
本模块支持4类核心输入源，所有输入均支持离线处理，具体规范如下：

| 输入类型 | 输入源 | 数据格式要求 | 核心处理规则 |
| :--- | :--- | :--- | :--- |
| 上游节点联动输入 | DAG画布中文本节点、图片节点的输出端口 | 标准化文本/图片资产ID格式，与全局资产体系对齐 | 自动读取上游节点的剧本文本、角色参考图、全局风格变量，作为解析核心输入，上游内容变更时触发增量解析 |
| 手动输入内容 | 脚本节点控制面板的手动输入框 | 纯文本格式，支持剧本、剧情大纲、创意描述 | 自动识别文本中的场景、角色、对白、动作标注，完成结构化拆分 |
| 文档导入输入 | 用户本地上传的TXT/Word/Excel文档 | 支持常见办公文档格式，自动提取纯文本内容 | 自动过滤文档格式标签、空行、无效符号，标准化剧本格式，适配专业剧本格式与普通文本格式 |
| 参考素材反向解析输入 | 用户本地上传的视频/分镜图集 | 本地视频文件、常见格式图片文件 | 调用本地多模态大模型，解析素材的镜头、画面、节奏、对白，反向生成分镜脚本 |

### 4.2 输出规范
本模块输出标准化的结构化数据，100%兼容节点体系与下游生成节点，核心输出内容分为6大部分，具体规范如下：

| 输出模块 | 核心字段 | 数据格式要求 | 下游消费场景 |
| :--- | :--- | :--- | :--- |
| 分镜列表核心数据 | 镜号、幕号、单镜时长、画面描述、景别、光影氛围、音效、对白、运镜方式、分镜唯一shotId | 数组格式，每个分镜为一个独立对象，字段非空校验，镜号连续不重复 | 分镜可视化渲染、下游生成节点的核心输入、时间轴视图渲染 |
| 全局角色库数据 | 角色唯一roleId、角色名称、角色固定描述、角色图资产ID、人设特征、全局风格约束 | 数组格式，每个角色唯一ID标识，全片通用 | 全片人物IP锁定、下游生成节点的人物一致性约束、角色库管理 |
| 分镜内角色实例数据 | 分镜shotId、绑定角色roleId、本镜角色动作、本镜角色情绪、本镜角色对白、本镜角色站位 | 与分镜列表一一绑定，通过shotId+roleId关联 | 分镜画面提示词生成、配音情绪匹配、角色动作细节生成 |
| 标准化提示词数据 | 分镜shotId、分镜画面完整提示词、视频运动完整提示词 | 严格遵循预设的固定模块化格式，与分镜一一绑定 | 下游图片/视频生成节点的直接输入，无需二次修改 |
| 全局时序坐标系数据 | 分镜shotId、入点时间timeIn、出点时间timeOut、全片总时长、分镜总数 | 时间单位为秒，精确到1位小数，时序连续无重叠、无断点 | 下游音视频合成节点的音画同步、时间轴视图渲染、节奏分析 |
| 解析元数据 | 解析模板、使用模型、解析时间、版本号、修改标记、人工锁定标记 | 结构化格式，与项目绑定 | 增量解析判断、版本回滚、操作追溯、人工编辑防覆盖 |

## 五、核心流程详细设计
本模块的核心执行流程分为9个标准步骤，全流程可离线运行，支持断点续跑、增量解析、人工干预，详细设计如下：

### 步骤1：解析触发与参数初始化
1.  **触发条件**：支持3种触发方式，均由DAG引擎统一调度
    - 自动触发：上游节点执行完成后，DAG引擎自动触发本模块执行
    - 手动触发：用户在脚本节点控制面板点击「生成分镜」按钮手动触发
    - 增量触发：用户修改分镜核心参数/上游输入内容变更后，触发增量解析
2.  **参数初始化**：执行前自动加载用户选择的解析模板、Skill技能规则、Prompt格式规范、模型配置、全局风格变量，完成运行环境初始化
3.  **状态同步**：执行开始前，将脚本节点状态更新为「执行中」，锁定节点参数与端口，禁止修改，同步状态给DAG引擎与前端UI

### 步骤2：多源输入归一化预处理
1.  **多源输入合并**：自动读取所有有效输入源，按优先级合并为统一的待解析文本，优先级：手动输入＞上游节点输入＞文档导入输入
2.  **格式清洗**：自动去除输入内容中的多余空行、特殊符号、格式标签、乱码内容，统一换行符、标点符号、对白标注格式
3.  **内容结构化拆分**：自动识别并拆分剧本中的场景描述、角色对白、动作标注、旁白、镜头提示，剔除无效的文学化、抽象化表述，保留可可视化的核心内容
4.  **无效内容过滤**：自动过滤违规内容、敏感词、无意义的填充内容，给出合规风险提示，同时过滤过短、无有效信息的输入内容
5.  **输出**：生成标准化的纯文本剧本，作为后续解析的唯一输入源

### 步骤3：语义解析与分镜结构化拆分
1.  **解析规则加载**：加载用户选择的Skill技能模板，读取模板内置的分镜密度规则、节奏配比、镜头拆分规范、输出格式约束
2.  **分镜粒度控制**：支持4种可切换的分镜拆分粒度，用户可一键切换：
    - 句子级：每一句对白/剧情描述生成1个分镜（适配强钩子短剧）
    - 场景级：每一个场景/剧情段落生成1个分镜（适配常规叙事）
    - 时长级：按用户设定的固定时长均匀拆分分镜（适配广告/卡点视频）
    - 自定义级：按用户指定的单镜时长/单幕分镜数量拆分
3.  **大模型语义解析**：调用本地部署的大语言模型/离线兼容的云端API，基于预设的系统提示词+Few-Shot示例，完成剧本的语义解析，按预设粒度拆分为独立分镜，同时提取每个分镜的核心字段：景别、运镜、光影、音效、对白等
4.  **分镜内容补全**：基于剧本上下文，自动补全分镜的缺失信息，比如场景环境、镜头逻辑、画面细节，保证每个分镜的画面描述可直接用于AI生成
5.  **输出**：生成初步的分镜列表数组，包含所有分镜的基础字段

### 步骤4：全局角色体系构建
1.  **角色提取**：从剧本与解析结果中，自动提取所有出场角色，生成基础角色档案，包含角色名称、外貌特征、服装人设、出场场景
2.  **全局角色库生成**：为每个角色分配全局唯一的roleId，去重合并同一角色的不同描述，生成全片通用的固定角色特征，避免同一角色在不同分镜中描述不一致
3.  **角色IP特征锁定**：支持用户上传角色参考图，自动提取人物核心特征，生成标准化的人物特征描述，写入全局角色库，同时绑定唯一资产ID，全片所有分镜共享
4.  **分镜角色实例绑定**：为每个分镜绑定对应的全局角色，生成分镜内的角色实例，补充该分镜内的角色动作、情绪、对白、站位，不修改全局角色的基础设定
5.  **输出**：生成全局角色库数组，以及与分镜一一绑定的角色实例数据

### 步骤5：固定格式提示词自动生成
1.  **规则引擎加载**：加载PromptPilot规则引擎，读取预设的固定提示词骨架、模块顺序、分隔符规则、字段映射关系
2.  **分镜画面提示词生成**：严格遵循用户预设的固定模块化格式，将分镜的每个字段自动填充到对应的提示词模块中，自动替换全局风格变量、角色特征变量，生成完整的、格式完全统一的分镜画面提示词
3.  **视频运动提示词生成**：基于预设的运动提示词骨架，自动提取分镜的运镜、动作、时长、环境动态信息，填充到对应模块，生成标准化的视频运动提示词
4.  **人工编辑防覆盖**：标记用户手动修改过的提示词模块/分镜，重新生成时不覆盖人工编辑的内容，仅更新未锁定的模块
5.  **输出**：与分镜一一绑定的完整画面提示词、视频运动提示词

### 步骤6：全局时序坐标系自动计算
1.  **基础时序计算**：基于每个分镜的单镜时长，自动累加计算全片时序，生成每个分镜的入点时间timeIn、出点时间timeOut，保证时序连续无重叠、无断点
2.  **时序联动更新**：用户修改单个分镜的时长、顺序时，自动重算全片所有分镜的入点/出点，保证时序连贯
3.  **全片统计计算**：自动计算全片总时长、分镜总数、平均单镜时长、节奏分布数据
4.  **输出**：每个分镜的时序数据、全片时序统计信息

### 步骤7：全量数据格式校验与容错处理
1.  **Schema强校验**：基于预设的分镜全字段Schema，对所有输出数据做全量校验，检查必填字段是否非空、数据类型是否匹配、镜号是否连续、时序是否连贯、提示词格式是否符合规范
2.  **分级容错处理**：
    - 轻微错误：单分镜字段缺失、格式不规范，自动补全修复，不中断解析流程
    - 中度错误：单分镜解析失败、提示词格式错误，自动针对该分镜重试解析，最多重试3次，失败则标记异常，不影响其他分镜
    - 严重错误：核心输入为空、模型调用失败、全量格式错误，中断解析，弹出明确的错误原因与解决方案
3.  **人工锁定内容保护**：校验过程中，对用户标记为「人工编辑锁定」的分镜/字段，不做任何修改与覆盖，仅做格式合规性提示
4.  **输出**：校验通过的标准化全量分镜数据，以及异常分镜标记与错误信息

### 步骤8：解析结果本地持久化与版本管理
1.  **数据持久化**：将校验通过的全量分镜数据，完整写入本地项目文件，加密存储在用户本地磁盘，不上传到任何云端
2.  **版本快照生成**：每次解析完成后，自动生成项目快照，保存解析前的原始数据、解析后的全量数据、解析参数、版本号，支持一键回滚
3.  **增量更新记录**：记录本次解析相对上一版本的变更内容，标记新增、修改、删除的分镜，支持变更对比
4.  **自动备份**：将解析结果自动备份到本地备份目录，避免项目文件损坏导致的数据丢失

### 步骤9：状态同步与数据输出
1.  **节点状态更新**：解析完成后，将脚本节点状态更新为「执行成功」，解锁节点参数与端口，同步状态给DAG引擎与前端UI；解析失败则更新为「执行失败」，同步错误信息
2.  **前端UI渲染**：将解析结果同步给脚本节点的双模块，在表格视图、时间轴视图中实时渲染分镜内容，支持用户可视化编辑
3.  **下游节点数据推送**：通过节点输出端口，将标准化的分镜数据、提示词、时序数据、角色数据，同步给下游关联的图片/视频/TTS/合成节点，触发下游节点的就绪状态

4.  **模型资源本地管理**：所有模型文件均存储在用户本地磁盘，模块自动扫描本地模型目录，支持模型一键切换、参数配置、性能优化

### 7.2 无网络环境降级策略
1.  **核心功能全离线可用**：无网络环境下，模块的核心解析、分镜生成、提示词生成、时序计算功能100%可用，仅依赖本地大模型
2.  **功能平滑降级**：无网络环境下，自动关闭云端API相关功能，保留本地核心能力，不影响主流程使用，同时给出明确的网络状态提示
3.  **本地资源预加载**：所有解析模板、规则、示例、内置词库，均预打包在软件本地安装包中，无需联网下载，保证离线环境下功能完整

1.  **全量数据本地存储**：所有用户剧本、分镜数据、项目文件、解析结果，100%加密存储在用户本地磁盘，模块不会上传任何用户数据到云端

3.  **本地权限最小化**：模块仅申请必要的本地文件读写权限，不读取用户无关的磁盘数据，保护用户隐私
4.  **无用户数据收集**：模块不收集用户的剧本内容、创作数据、使用行为，完全符合单机软件的隐私保护要求

## 八、与其他模块的联动规则
### 8.1 与DAG工作流引擎的联动
1.  模块的执行生命周期完全由DAG引擎调度，严格遵循拓扑排序结果执行，上游节点未执行完成时，不会触发解析流程
2.  模块执行过程中，实时将节点状态、执行进度、错误信息同步给DAG引擎，接受引擎的统一管控
3.  支持DAG引擎的断点续跑、暂停/继续、增量执行调度指令，适配大项目的批量执行需求

### 8.2 与脚本节点双模块的联动
1.  与上方资产容器模块：模块的解析结果完整写入资产容器模块，作为节点的核心资产持久化存储；同时从资产容器模块读取上游输入数据、项目配置、资产ID
2.  与下方控制面板模块：从控制面板读取用户设置的解析参数、模板选择、模型配置、规则开关；同时将执行进度、解析结果、错误信息实时同步到控制面板，实现可视化交互

### 8.3 与下游生成节点的联动
1.  模块输出的标准化数据，通过节点输出端口，100%兼容下游图片/视频/TTS/合成节点的输入规范，下游节点可直接使用，无需二次修改
2.  分镜唯一shotId全链路绑定，下游节点生成的内容均绑定对应shotId，实现分镜-成片一键回溯、单分镜重跑
3.  时序锁机制联动，修改分镜时序数据后，自动同步给下游音视频节点，保证全片音画同步

### 8.4 与本地项目管理模块的联动
1.  解析结果自动同步到项目管理模块，纳入项目全生命周期管理
2.  自动生成项目快照与备份，支持项目版本管理、回滚、归档
3.  支持项目文件的导入导出、跨项目模板复用

## 九、异常处理与容灾保障
### 9.1 分级异常处理机制
| 异常等级 | 异常场景 | 处理策略 | 兜底方案 |
| :--- | :--- | :--- | :--- |
| 轻微异常 | 单分镜字段缺失、格式不规范、提示词模块缺失 | 自动修复，不中断流程，完成后给出提示 | 保留用户原始输入，支持手动修改 |
| 中度异常 | 单分镜解析失败、模型调用超时、单分镜校验失败 | 自动重试3次，重试失败标记异常，继续执行其他分镜 | 保留已完成的分镜结果，支持单分镜手动重跑 |
| 严重异常 | 核心输入为空、模型加载失败、全量格式校验失败 | 中断执行，弹出明确的错误原因与解决方案 | 保留上一版本的有效解析结果，支持一键回滚 |
| 致命异常 | 软件崩溃、断电、磁盘写入失败 | 崩溃前自动触发紧急备份，每30秒写入临时缓存 | 重启软件后自动弹出项目恢复窗口，一键恢复到崩溃前状态 |

### 9.2 容灾与数据保障机制
1.  **实时热备份**：用户每修改一个分镜，自动将最新内容写入本地IndexedDB热备份，延迟不超过500ms，保证崩溃后可恢复到最新编辑状态
2.  **定时冷备份**：每5分钟自动生成项目完整快照，压缩后写入本地磁盘，最多保留30个历史快照，支持一键回滚到任意时间点
3.  **断点续跑机制**：解析过程中断后，重新执行时自动跳过已完成的分镜，从失败的位置继续执行，无需全量重新解析
4.  **项目文件修复**：针对意外断电、崩溃导致的项目文件损坏，内置项目修复工具，可提取文件中的有效分镜数据，最大程度恢复用户创作内容
