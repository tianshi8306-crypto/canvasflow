# CanvasFlow AI Studio 软著材料格式说明

## 一、操作说明书转 Word 步骤

1. 用 Microsoft Word 或 WPS 打开 `CanvasFlow-AI-Studio-软件操作说明书.md`（或直接复制全文粘贴到 Word）
2. 设置页面格式（软著常用规范）：
   - 纸张：A4
   - 页边距：上下 2.54 cm，左右 3.17 cm
   - 正文字体：宋体，小四号（12pt）
   - 行距：1.5 倍行距
   - 章标题：黑体，三号，段前段后 0.5 行
   - 节标题：黑体，四号
3. 在封面页填写：
   - 软件全称：**CanvasFlow AI Studio**
   - 版本号：**V0.5.0**
   - 文档类型：操作说明书
   - 著作权人：（填写您的单位或个人名称）
4. 生成目录：Word → 引用 → 目录 → 自动目录

## 二、页数说明

当前说明书正文为 **V2.1**，约 **27,000 字**，按小四号宋体、1.5 倍行距排版后约 **28～32 页**。文中 **【截图占位 · 图 X-Y】** 共 28 处，插入实拍截图后可稳定达到并超过 30 页。说明书仅描述本软件功能，不含其他产品名称或源代码路径。

若页数不足 30 页，可：
- 在第四、六、十、十五、十七章插入软件截图（每章 2～3 张）
- 将部分表格改为图文混排

若页数超过 40 页，可：
- 缩小页边距至 2 cm
- 删除附录 F 版本历史中较早版本条目

## 五、其他软著材料提醒

除操作说明书外，软著申请通常还需要：

1. **源代码**：前后各连续 30 页（共 60 页），每页不少于 50 行
   - 前端建议取：`src/components/FlowCanvas.tsx`、`src/store/projectStore.ts` 等
   - 后端建议取：`src-tauri/src/executor/engine.rs`、`src-tauri/src/executor/script_node.rs` 等
2. **申请表**：在中国版权保护中心官网在线填写
3. **身份证明**：个人身份证或企业营业执照复印件

### 重新生成 Word 与源代码

在项目根目录执行：

```bash
npm install --no-save docx
node scripts/generate-softcopyright-docs.mjs
```

生成规则：
- **操作说明书**：由 Markdown 转 Word，含封面、页眉（软件名+版本号）、页码
- **源代码**：从 35 个核心源文件按业务顺序拼接，提取连续前 30 页与后 30 页（每页 50 行）
