# UI / 体验迭代指南（低复杂度）

与 [路线图 V2](../iterations/ROADMAP_V2.md) 中「创作体验层 / 设计与体验」子维度对应。目标是把界面计划写进**迭代执行单**，可验收、可追溯，而不引入独立 Design System 或强制 Figma。

## 1. 样式与真源

- **真源**：[`src/styles/global.css`](../../src/styles/global.css)（语义变量、`.nodeCard`、`.btn`、侧栏 `.panelBody` 等）。
- **约定**：新界面优先复用现有 class；若新增全局样式，集中写在 `global.css` 并简短注释用途。
- **非目标（默认）**：多主题皮肤、组件库发包、像素级设计标注，除非单独立项。

## 2. 迭代执行单中的 UI/UX 小节（推荐模板）

在每轮 `iteration-xx-*.md` 中增加一节 **「UI/UX」**（可与功能清单并列），至少回答：

1. **关键界面**：本轮涉及哪些屏区（画布 / 侧栏 / 弹层 / 顶栏等）。
2. **关键状态**：空、加载、成功、失败、禁用；禁用是否带 `title` 说明。
3. **键盘与焦点**：是否依赖快捷键；是否在 `INPUT`/`TEXTAREA`/可编辑区内避免误触全局快捷键。
4. **本轮 UI 非目标**（可选）：例如「不改品牌色」「不重排主导航」。

若本轮**无任何界面变更**，请显式写一行：**「本轮无 UI 变更」**，便于评审跳过视觉验收。

## 3. 按轮次的 UI 强度（建议）

| 轮次 | UI 强度 | 说明 |
|------|---------|------|
| R1 | 低 | 顶栏、状态、基础表单与错误文案即可。 |
| R2 | **高** | 节点卡、侧栏、画布菜单与拖拽反馈统一。 |
| R3 | 中高 | 脚本工作台表格/卡片、勾选与持久化反馈。 |
| R4 | **高** | 分镜结果列表/网格与回看布局。 |
| R5 | **高** | 参数面板分组、生成中/失败态。 |
| R6 | **高** | 时间线控件与导出进度。 |
| R7 | 中 | Provider、日志、费用等偏信息密度与可读性。 |
| R8 | **高** | 资产库浏览、筛选、预览与模板入口。 |
| R9 | 中 | 编排向导与发布前检查的信息架构。 |

「高」表示执行单中 **UI/UX 小节不可省略**（可简写，不可缺）。

## 4. 建议在「手工验收」之外增加的勾选项（2–5 条）

可按轮复制到执行单末尾：

- [ ] 空状态有说明文案，用户知道下一步做什么。
- [ ] 错误态不遮挡主操作区；关键错误在状态栏或固定区域可见。
- [ ] 主要按钮与危险操作（删除等）在视觉上可区分（已有 `btnDanger` 等则复用）。
- [ ] 在侧栏输入时，画布级快捷键不误伤（与 App 层焦点策略一致）。
- [ ] 浅色/深色下对比度可接受（当前为暗色单主题，以 `global.css` 为准）。

## 5. 与 Cursor 规则的关系

仓库规则要求：涉及主界面或新面板的迭代，执行单须含 **UI/UX 小节** 或 **「本轮无 UI 变更」**。见 `.cursor/rules/low-complexity-iteration.md`。
