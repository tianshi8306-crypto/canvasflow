# 里程碑 M5（D.1 + D.2 试点）

## 目标

- 让脚本镜头（`ScriptBeat.id`）可以直接驱动一条可执行的视频生成链路。
- 在不重构执行器大架构的前提下，提供一键可演示入口。

## 本次试点实现

- 在脚本分镜区新增按钮：`生成视频链路（试点）`。
- 行为：
  - 读取勾选镜头（无勾选则默认全部）；
  - 为每条镜头创建一个 `videoNode`；
  - 自动从脚本节点连线到视频节点（`scriptNode -> videoNode`）；
  - 将 `scriptBeatId`、`shotNumber` 写入视频节点 `data.params`；
  - 将分镜文案/描述 + 视频运动提示词拼接为视频草稿提示词（`video.draft.prompt`）。

## 验收建议

1. 在脚本节点填写/生成多条镜头并勾选 1~3 条。
2. 点击「生成视频链路（试点）」。
3. 验证画布右侧新增对应数量的视频节点，并已自动连线。
4. 打开任一视频节点，确认提示词已带入镜头内容；节点参数里可看到 `scriptBeatId` / `shotNumber`。

## 已知边界（后续迭代）

- 当前仍由前端视频桥接发起任务，尚未完全迁入 `executor` 算子化调度。
- 暂未把镜头时码/时间线合成联动到导出流程（D.1 深化项）。
